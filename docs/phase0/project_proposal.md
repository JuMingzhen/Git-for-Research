# 项目提议文档

## 具体目标

构建一个面向导师-学生课题组的学术协作系统，用“项目主线 + 个人分支 + 组会节点”组织科研过程。系统输出项目总览、个人进展、组会纪要、任务回流与历史问答，帮助团队同步信息。

## 核心价值

它针对课题组常见的进度不透明、组会前后信息断裂和任务靠人工整理等低效问题。价值在于把零散科研活动变成可追踪结构，并用 AI 自动完成总结、拆分和提醒，减少机械记录负担，提高协作效率。

## 可行性简析

第一步先做文本版 Web demo，跑通“学生更新 -> 导师会前查看 -> 组会总结 -> 任务回流 -> 历史问答”闭环。所需资源是前后端开发、示例数据和 LLM 接口。主要风险是 AI 输出不稳定与需求扩张；验证方式是用真实组会流程做小范围演示，看是否显著提升信息同步效率。
