# MVP 功能边界

这个文件专门回答一个问题：**第一版到底做什么，不做什么。**

## 第一版必须完成

### 1. 研究分支结构

- 导师创建项目
- 系统展示主分支
- 每个学生有自己的个人分支
- 学生可以创建子分支

### 2. 日常进度管理

- 学生提交本次进展
- 记录 blocker 和 next step
- 查看某个分支的历史更新
- AI 根据更新生成阶段总结

### 3. 组会协作

- 创建组会节点
- 录入文本会议记录
- AI 生成会前 briefing
- AI 生成会后纪要
- AI 从纪要中抽取任务并关联到学生分支

### 4. 历史问答

- 基于会议和进展文本做检索式问答
- 返回答案时展示来源片段

### 5. 基础展示能力

- 项目总览页
- 分支详情页
- 进展更新页
- 组会页
- 任务页
- 问答页

## 第一版刻意不做

- 语音输入与自动转录
- 图片、PDF、实验图表等多模态输入
- 复杂组织权限与学院级管理
- 真正的多 Agent 协作
- 外部平台联动
- 自动生成科研内容本身

## 为什么这样裁剪

因为这个项目的最大风险不是“技术不够炫”，而是“范围太大导致核心闭环做不出来”。  
第一版只要能证明下面这条闭环成立，就已经是一个合格 demo：

`学生更新 -> 导师会前了解背景 -> 组会后任务自动回流 -> 历史内容可追问`

## 正式开发时的建议优先级

1. 分支结构与进展更新
2. 组会总结与任务回流
3. 问答检索
4. 周报整理
5. Agent 扩展接口
